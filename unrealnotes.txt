Treat macros as code generators, not logic executors.

    Use Functions for game logic, math, and anything that needs to happen while the game is running.

    Use Macros (or rather, use Unreal's built-in macros) when you need to strip code out of final builds (like debug lines), 
    handle cross-platform boilerplate, or do things that require information about the source file itself.

    Unreal C++ Macros vs. Functions

    The Core Difference: Macros are literal copy-paste text replacements that happen before the code even compiles. Functions are compiled blocks of logic that execute at runtime.

    Type Safety: Functions are strictly type-checked (just like your C# methods). Macros completely bypass type checking because they just blindly paste text.

    Debugging: You can step into a function line-by-line in Visual Studio or Rider. You cannot step into a macro.

    Performance: Macros have zero function-call overhead (because the code is injected directly into the line), but heavy use can bloat your compiled file size.

    Unity/C# Equivalent: There isn't one. C# #define is only for conditional compilation (#if UNITY_EDITOR), whereas Unreal C++ macros (UE_LOG, TEXT()) are literally writing or formatting code for you behind the scenes before the compiler steps in.
